- Too many ideas
- Never see them through completion
- Inconsistent execution

**This ends now.**

## 📜 The Five Rules

### Rule 1: One Project at a Time

**Active Project:** AgentHub Beta Launch

**Frozen Until Completion:**
- Card Tribe (aborted)
- New marketplace ideas
- New content frameworks
- Any other "shiny object"

**Completion Criteria for AgentHub:**
- [ ] 10 recruitment emails sent
- [ ] 20 responses tracked
- [ ] 5 users onboarded
- [ ] $1 revenue (proof of concept)

**Only after ALL boxes checked → Next project allowed.**

---

### Rule 2: The 48-Hour Rule

**New ideas must wait 48 hours before action.**

**Process:**
1. Idea arrives
2. Write to `memory/Backlog-Ideas.md`
3. Close file
4. Don't touch for 48 hours
5. After 48 hours: Add to backlog (not active) or discard

**Backlog is not a to-do list. It's a graveyard for most ideas.**

---

### Rule 3: Weekly Accountability Check-In

**Every Friday at 6 PM SGT:**

**Subhuti asks:**
1. What did I ship this week?
2. What's blocked?
3. What will I ship next week?

**Michael answers honestly. No excuses.**

**If no response → Subhuti follows up next week.**

---

### Rule 4: The "Done" List

**Every Friday, write to `memory/YYYY-MM-DD.md` (Friday's file):**

```markdown
## Today's Completions

- [ ] Email sent: ___
- [ ] Content published: ___
- [ ] User onboarded: ___
- [ ] Revenue generated: $___
- [ ] Other output: ___
```

**No "worked on." Only "done."**

**If list is empty → You had an unproductive day.**

---

### Rule 5: Public Commitment

**Once per week, public declaration:**

Post on LinkedIn/Twitter:
> "Launching [X] by [date]. Ask me if it's done."

**Let the market hold you accountable.**
