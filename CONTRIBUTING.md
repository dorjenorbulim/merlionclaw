# Contributing to MerlionClaw

Thank you for your interest in contributing! 🦁

## How to Contribute

### 1. Submit a Use Case Story

Share how you use OpenClaw to enrich your life or business.

**Requirements:**
- Must be your own experience or properly attributed
- Must meet our [ethical framework](../index.html#ethical-use)
- Must be legal under Singapore law
- Can be anonymous if preferred

**Template:** Copy `stories/hawker-automation.html` and modify.

### 2. Submit a Skill

Built an OpenClaw skill? Share it!

**Requirements:**
- Code must be your own or properly licensed
- Include clear documentation
- No malicious functionality

### 3. Improve the Site

- Fix bugs
- Improve accessibility
- Add features
- Translate content

## Submission Process

1. **Fork** the repository
2. **Create a branch** (`git checkout -b feature/my-feature`)
3. **Make changes** following our style
4. **Test locally** (open HTML files in browser)
5. **Commit** (`git commit -m "Add feature"`)
6. **Push** (`git push origin feature/my-feature`)
7. **Open a Pull Request**

## Review Process

- All submissions are reviewed for ethical compliance
- We may request changes
- Typical review time: 3-5 days

## Code of Conduct

- Be respectful and constructive
- No spam or self-promotion
- Focus on helping the community

## Questions?

Open an issue or email hello@merlionclaw.sg
